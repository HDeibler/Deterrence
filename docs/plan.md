# Deterrence Next-Phase Plan

## Purpose

This project is evolving from a missile-and-radar visualization into a continuous-time grand-strategy simulation where military power is generated by industry, trade, logistics, basing, and political access.

The goal is not to let the player freely spawn force anywhere on Earth. The goal is to make military action the downstream consequence of:

- domestic production
- foreign resource access
- transport capacity
- hub basing
- forward deployment
- diplomacy
- war damage to supply chains

That is what creates realism, restraint, escalation pressure, and long-horizon strategic reward.

## Current Foundation

The current game already has:

- playable major powers: `USA`, `CHN`, `RUS`
- Earth map, borders, country selection, and god-view toggles
- military installation dataset and base overlays
- missile launches, ballistic trajectories, and predicted flight paths
- radar placement and early-warning satellite launch/orbit mechanics
- naval asset scaffolding

The next phase should preserve that military layer and place it inside a broader economy-and-logistics simulation.

## Design Principles

### 1. Continuous-Time Simulation

The game should not be turn-based.

Everything runs on one simulation clock with different system cadences:

- render: every frame
- tactical systems: missiles, radar, intercepts
- operational systems: shipments, rebasing, launches
- strategic systems: production, trade, diplomacy, AI

This preserves continuity and lets consequences emerge from prior decisions rather than from turn resolution.

### 2. No Instant Strategic Projection

Assets should not appear in a theater because the player clicked there.

Military capability should only reach a region if the country has:

- produced the asset
- moved the asset
- sustained the asset
- secured permission to base it
- defended the logistics chain supporting it

### 3. Strategic Systems Must Drive Tactical Systems

Missiles, ships, aircraft, radars, satellites, and future interceptors should be outputs of:

- resources
- industrial capacity
- transport capacity
- basing access

The military map is the visible result of invisible strategic systems.

### 4. Scale Through Abstraction

The simulation should not model every truck, worker, or barrel of oil.

Scale should come from layered abstraction:

- stockpiles and rates for the global economy
- route flows and selected shipment entities for logistics
- explicit units only for player-visible or contested military objects

## Game Vision

Each major power starts with baseline domestic capacity to produce:

- oil throughput
- rare-earth extraction
- chips
- military industrial output

Foreign access does not create production from zero. It increases capacity, removes bottlenecks, and improves resilience.

Third-party countries produce strategic resources and control geographically important locations. The player must negotiate with AI-run countries for:

- resource contracts
- port access
- hub basing rights
- transit rights
- security guarantees
- exclusive deals

Once resources flow into the player's economy, military production increases. Produced assets are then transported from homeland reserves to global hub bases and from hubs to real-world forward bases.

War should disrupt this chain. If a major producer attacks mines, ports, chip factories, or hub bases, the effects should propagate into stockpiles, industrial output, and operational readiness.

## Core Systems

The next phase needs six new runtime systems.

### 1. Resource System

Responsible for:

- deposits
- extraction sites
- depletion later
- domestic baseline production
- foreign import flows

First-pass resources:

- oil
- rare earths
- chips

Derived industrial outputs:

- ship production capacity
- aircraft production capacity
- missile production capacity
- rocket production capacity

### 2. Production System

Responsible for:

- production queues
- recipes
- industrial throughput
- stockpile consumption
- output inventories

Suggested first recipes:

- rare earths -> chips
- oil + chips + factory capacity -> cruiser
- oil + chips + factory capacity -> submarine
- oil + chips + factory capacity -> radar
- oil + chips + factory capacity -> missile inventory
- oil + chips + factory capacity -> cargo plane
- oil + chips + factory capacity -> oil ship
- chips + oil + rocket capacity -> launch vehicle
- launch vehicle + radar payload + chips -> early-warning satellite

### 3. Logistics System

Responsible for:

- trade routes
- shipping lanes
- airlift lanes
- hub throughput
- base throughput
- route vulnerability
- ongoing sustainment

This system is central to the game.

Military assets in hubs and forward bases should generate constant logistical demand. Oil, spare parts, missiles, and general sustainment should be consumed over time simply by maintaining force posture.

### 4. Deployment System

Responsible for:

- movement from national reserve to hub
- movement from hub to forward base
- rebasing restrictions
- sticky forward deployment

This is where the player's global posture becomes visible on the map.

### 5. Diplomacy / Contract System

Responsible for:

- resource contracts
- port access
- transit rights
- hub rights
- alliance pressure
- sanctions
- coercion
- embargoes

This should be deterministic and rule-driven first. Natural-language AI can be added later as a presentation layer for negotiation, not as the underlying decision engine.

### 6. Strategic AI System

Responsible for how third-party countries and rival powers react to:

- offers
- threats
- military buildup
- shortages
- strikes on infrastructure
- sanctions
- alliance changes

Each country should evaluate profit, security, alignment, vulnerability, and leverage.

## Hub Bases And Forward Bases

This distinction is critical.

### Hub Base

A hub base is a strategic warehouse and theater-entry node.

It should:

- store large oil reserves
- store large missile reserves
- store large aircraft reserves
- receive bulk logistics by sea and air
- distribute assets regionally
- provide high ingress and egress throughput

### Forward Base

A forward deployed real base is an operational endpoint.

It should:

- have much smaller storage
- hold mission-ready assets
- support local sorties, launches, and deployments
- have decent inbound throughput
- have poor outbound throughput

### Sticky Deployment

Assets moved into a forward base should be "sticky."

That means:

- moving them in is relatively fast
- moving them out is slow and expensive
- while stationed there, they consume local support and logistics
- a damaged or isolated forward base becomes a serious liability

This prevents unrealistic global shuffling of force posture.

## Transport Assets

Transport should become a real military-economic asset category.

### Cargo Plane

Used for:

- missiles
- chips
- radar components
- urgent resupply
- selective aircraft support packages

Properties:

- fast
- limited capacity
- expensive
- useful for urgent reinforcement and high-value cargo

### Oil Ship / Tanker

Used for:

- bulk oil movement
- sustained industrial and naval support

Properties:

- slow
- large capacity
- vulnerable to naval interdiction and blockade
- essential for large-scale sustained projection

These transport assets should themselves be produced, assigned, and attrited.

## Economic Model

The game should expose both money and materials.

### Government Finance View

The player should see:

- tax income
- treasury / reserve balance
- export revenue
- import cost

And a breakdown of expenses:

- operational cost
- logistical cost
- development cost
- procurement cost
- basing cost

### Resource View

The player should also see real material flows:

- oil stockpile
- oil inflow rate
- oil consumption rate
- rare-earth stockpile
- rare-earth inflow rate
- chip stockpile
- chip production rate
- ship production rate
- plane production rate
- missile production rate
- rocket production rate

The important UI concept is derivative and trend, not just static quantity. The player should be able to tell whether they are net positive or net negative in each strategic material and industrial output.

## War And Supply Chain Effects

War should damage systems, not only units.

Strategic targets should include:

- oil fields
- rare-earth mines
- chip factories
- asset factories
- ports
- hubs
- spaceports
- naval logistics lanes
- cargo aircraft routes

Example:

If China strikes rare-earth production in Australia:

- rare-earth export capacity falls
- chip production drops in importing countries
- output of radars, missiles, aircraft, ships, and rockets falls
- contracts get repriced or cancelled
- stockpiles drain faster than they replenish
- countries change diplomatic behavior

This is a core intended gameplay effect, not an edge case.

## Recommended Phasing

### Phase 1: Strategic Economy MVP

Build:

- baseline domestic production for `USA`, `CHN`, `RUS`
- national stockpiles
- national reserve and domestic base allocation
- chip factory and asset factory queues
- player factory placement on the world map
- financial income / expense breakdown
- local save / resume flow
- tactical consumption of strategic inventory

Goal:

Make the domestic economy visible, continuously changing, and directly tied to tactical actions.

### Phase 2: Contracts And Imports

Build:

- third-party oil and rare-earth producers
- trade agreements
- import routes
- resource inflow from third-party states
- deterministic negotiation AI

Goal:

Make foreign dependence real.

### Phase 3: Hub Logistics

Build:

- hub base creation
- strategic storage
- route throughput
- cargo plane and oil ship production
- homeland -> hub movement

Goal:

Make global projection require infrastructure and transport.

### Phase 4: Forward Deployment

Build:

- hub -> real base transfer
- sticky forward deployment
- base storage and throughput
- maintenance and local consumption

Goal:

Make the real-world military dataset operationally meaningful.

### Phase 5: Disruption And War Economy

Build:

- strikes on mines, factories, ports, hubs, and routes
- supply shock propagation
- emergency rerouting
- sanction and embargo responses

Goal:

Make war reshape the economy instead of only destroying units.

### Phase 6: Radar / Intercept Integration

Build:

- radar-fed perception
- interceptor production and deployment
- consumption of logistics and chips by defense networks

Goal:

Merge the existing missile/radar gameplay with the strategic economy.

## State Authority

The project needs an explicit authority boundary before adding more logistics and factory management features.

### Current Practical Rule

For the current MVP:

- the server is authoritative for bootstrap seed data
- the client is authoritative for the live campaign simulation
- save data is the authoritative persisted snapshot of the live campaign

This is acceptable for the current phase because the game is still single-player and local-first.

### Required Future Rule

As logistics, contracts, and war-economy effects expand, the simulation should be organized around a stable campaign snapshot model rather than around ad hoc UI state.

Suggested top-level persisted state:

- `CampaignState`
- `StrategicState`
- `BaseNetworkState`
- `FactoryState`
- `TacticalState`

The key point is that all runtime systems should derive from and serialize back into these stable state objects rather than owning incompatible local truth.

## Base Normalization Rule

The real-world military installation dataset is source data, not the final gameplay model.

Implementation rule:

- installations remain the canonical geographic source
- gameplay bases are normalized runtime entities derived from installations
- capability rules determine what each derived base can store, deploy, and sustain

This avoids binding every gameplay mechanic directly to raw installation categories.

### Practical Interpretation

The existing installation dataset should map into gameplay nouns such as:

- `Base`
- `Hub`
- `ForwardBase`
- `Spaceport`
- `SiloBase`

Those normalized entities should then own:

- storage compatibility
- deployment compatibility
- sustainment cost
- throughput
- capacity

That mapping layer is mandatory before large-scale deployment and logistics features are added.

## Economy To Force Chain

The intended chain of authority should be explicit:

- `IndustrialFacility -> NationalReserve -> Hub -> ForwardBase -> TacticalUse`

For the current MVP, the implemented subset is:

- `IndustrialFacility -> NationalReserve -> DomesticBase -> TacticalUse`

This is the correct simplification for the current phase. Cross-border movement and hub logistics should extend this chain later rather than replacing it.

## Factory Management Before Full Logistics

There is a useful intermediate step between simple factory placement and full logistics gameplay.

Before adding hub logistics, selected industrial facilities should support constrained management actions such as:

- upgrade throughput
- pause production contribution
- reassign output emphasis

This preserves momentum in the strategic layer without prematurely introducing shipment complexity.

## Success Criteria

This phase succeeds when the player can clearly feel:

- resource scarcity
- industrial tradeoffs
- the value of foreign relationships
- the necessity of hub basing
- the cost of global posture
- the long tail of supply-chain disruption

The player should be able to lose a crisis because they failed to secure resources and logistics before the first missile launch.

That is the intended strategic identity of the game.
